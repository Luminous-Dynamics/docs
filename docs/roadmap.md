## Luminous Nix Roadmap

### Current Version: 1.0.x (Stable)

**Status:** Production-ready, actively maintained

### Q1 2026: Enhanced Intelligence (v1.1.0)

**Theme:** Better understanding, smarter execution

**Features:**
- **Context awareness:** Remember previous commands and learn from your patterns
- **Multi-step workflows:** Handle complex tasks like "set up a web dev environment"
- **Interactive clarification:** Ask questions when intent is ambiguous
- **Custom LLM providers:** Support for more AI backends (Anthropic Claude, local models)

**Developer Experience:**
- Plugin marketplace for community extensions
- Better debugging tools and logging
- Performance profiling

**Timeline:** March 2026

---

### Q2 2026: Collaboration & Scale (v1.2.0)

**Theme:** Team features and enterprise readiness

**Features:**
- **Multi-user support:** Shared configurations for teams
- **Audit logging:** Track all system changes
- **Role-based permissions:** Control who can do what
- **Remote management:** Manage multiple machines from one interface

**Enterprise:**
- SSO integration (SAML, OAuth)
- Compliance reporting
- SLA guarantees
- Dedicated support

**Timeline:** June 2026

---

### Q3 2026: Intelligence Expansion (v1.3.0)

**Theme:** Beyond NixOS, broader capabilities

**Features:**
- **Cross-platform expansion:** Support for Arch, Ubuntu, Fedora
- **Infrastructure as code:** Generate Terraform, Ansible from natural language
- **Cloud provider integration:** AWS, GCP, Azure management
- **Container orchestration:** Kubernetes management via natural language

**AI Improvements:**
- Fine-tuned models for system administration
- Faster local models (< 1 second response)
- Offline capability improvements

**Timeline:** September 2026

---

### Q4 2026: Community & Ecosystem (v2.0.0)

**Theme:** Open platform, community ownership

**Features:**
- **Decentralized plugin marketplace:** Community-built extensions
- **Shared knowledge base:** Learn from community patterns
- **Intent marketplace:** Share and discover workflows
- **Community governance:** Vote on feature priorities

**Architecture:**
- Core open-sourced (already MIT)
- Plugin API v2 (breaking changes, better design)
- Federation protocol (share intents securely)

**Timeline:** December 2026

---

### 2027 and Beyond

**Vision for mature Luminous Nix:**

- **Universal interface:** Manage any system through natural language
- **Proactive assistance:** "Your disk is 90% full. Should I clean up old builds?"
- **Learning system:** Adapts to your preferences automatically
- **Ecosystem:** Thousands of community plugins
- **Standard:** Natural language interfaces become expected, not novel

**Not planning to build:**
- âŒ Cloud-only service (sovereignty matters)
- âŒ Telemetry without consent (privacy first)
- âŒ Closed-source premium tier (all features open)
- âŒ Engagement-optimized features (consciousness-first)

- Does it respect attention?
- Does it amplify capability?
- Does it maintain transparency?
- Does it enable sovereignty?
- Does it serve wellbeing?

**If no, we don't build it.** Even if profitable.
